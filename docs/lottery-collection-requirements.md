# 答题补给与收藏图鉴系统 - 需求文档

> 状态：**头脑风暴稿 v1，已归档**。最新完整方案请见 [`miniapp-feature-plan-v2.md`](./miniapp-feature-plan-v2.md)。
> 立项动机：参考「乐观小🐑」专注小程序的 gacha 玩法，结合「申面智能小助手」已有用户体系，设计**学习激励 + 收藏成就**复合系统。
> 文档目标：在不破坏现有「额度制+绑号制」的前提下，增加**奖励侧**维度，提升用户停留时长和学习动力。

---

## 一、背景与目标

### 1.1 现状盘点
- 现有体系：额度制（每日 5 次免费 + 邀请码无限）+ 绑号制（Web/小程序共享）
- 现有玩法：AI 出题/批改、语音答题、真题/申论参考、政务要闻
- 现有激励：邀请码会员（无收集/展示元素）
- 现有缺失：**学习结果只有「扣额度」没有「奖励反馈」**，用户练习后没有可炫耀/可展示的成果

### 1.2 目标（三个层次）
| 层次 | 目标 | 衡量指标 |
|------|------|---------|
| 短期 | 给答题/专注加一个**轻量奖励反馈** | 完成练习后弹个「获得学习点+抽 1 次」 |
| 中期 | 形成**学习→奖励→收集→炫耀**闭环 | 收藏图鉴完成度、平均每日停留时长 |
| 长期 | 通过**图鉴分享**带来自然流量 | 分享率、新用户来源标注 |

### 1.3 核心判断
- **不要做成氪金 gacha**：本小程序是公考工具，抽奖应是**激励**而非**盈利**
- **不要干扰学习主流程**：抽奖是**完成练习后的奖励**，不是进入练习的门槛
- **要有公考特色**：奖池里不只有像素小动物，还要有「公考名师头像」「申论金句卡」等品类

## 二、核心玩法设计

### 2.1 三大模块关系图
```
┌─────────────────────────────────────────────────────┐
│  入口：首页 + 「我的」新增「补给站」入口              │
└─────────────────┬───────────────────────────────────┘
                  │
        ┌─────────┼─────────┐
        ▼         ▼         ▼
   ┌────────┐ ┌────────┐ ┌────────┐
   │ 专注台 │ │ 答题流 │ │ 补给站 │（新增）
   │(番茄钟)│ │(已有)  │ │(新页面)│
   └────┬───┘ └────┬───┘ └────┬───┘
        │          │          │
        ▼          ▼          ▼
   ┌─────────────────────────────┐
   │   产出「学习点」货币        │
   │   按答题/专注时长发放（见下文） │
   └─────────────┬───────────────┘
                 ▼
   ┌──────────────────────────────┐
   │  「补给站」消耗学习点抽奖        │
   │  3 学习点抽 1 次（4 档稀有度） │
   └─────────────┬────────────────┘
                ▼
   ┌──────────────────────────────┐
   │  获得补给品（6 大类）          │
   │  → 入「我的图鉴」/「公考补给站」│
   └──────────────────────────────┘
```

### 2.2 专注台（番茄钟）
**作用**：让用户主动承诺 30/60 分钟专注，结束后奖励学习点
**触发场景**：用户主动选择（不强制）

| 维度 | 设计 |
|------|------|
| 时长选项 | 30 / 60 分钟（预设 2 档）|
| 视觉 | 居中显示 1 只「当前 pixel pet」像素动物（从补给站抽到的），倒计时大字 |
| 暂停规则 | 最多暂停 1 次（避免用户拿专注时间刷分），恢复后继续倒计时 |
| 放弃规则 | 放弃则本时段不获得学习点，但允许开始下一段 |
| 完成奖励 | 30 分钟 +2 学习点 / 60 分钟 +4 学习点 |
| 异常处理 | 切后台 5 分钟以上自动放弃（防作弊）|

**与「每日任务 v2」的关系**：
- 每日任务（panda 状态机）：影响 mascot 形象，激励**每日首次练习**
- 专注台（新增）：影响学习点，激励**单次长时间练习**
- 两者独立，不冲突，可叠加

### 2.3 答题流（已有，叠加奖励）
**不改动现有答题流程**，仅在 4 个完成点增加奖励发放：
| 完成点 | 文件 | 学习点奖励 |
|--------|------|---------|
| AI 智能练习 - 单题批改完成 | `subpkg-practice/question/index.tsx` | +1 |
| 套题训练 - 整套批改完成 | `subpkg-practice/set-question/index.tsx` | +3 |
| 真题参考 - 我的作答完成 | `subpkg-zhenti/detail/index.tsx` | +1 |
| 申论训练 - 提交作答完成 | `subpkg-shenlun/detail/index.tsx` | +1 |

**关键约束**：原有额度扣减**不变**，学习点是**奖励维度**而非**替代维度**

### 2.4 补给站（新增）
**作用**：消耗学习点抽取补给品的核心玩法页面

| 维度 | 设计 |
|------|------|
| 入口 | 底部 Tab 暂不加（避免主结构改动），放「我的」页面 + 首页 Hero 卡 |
| 单次消耗 | 3 学习点/抽 |
| 免费抽 | 每日 1 次免费（每日 0 点刷新），免费抽不消耗学习点但产出固定为「普通」品质 |
| 动画 | 像素风格的「补给品盒」从屏幕外飞入 → 抖动 → 揭示（参考截图2的「?」盒子）|
| 概率（参考截图2）| 普通 70% / 稀有 23% / 史诗 6% / 传说 1% |
| 分享奖励 | 抽到的补给品可分享到微信好友/朋友圈，分享后**双方各 +1 学习点** |

## 三、补品池设计（核心差异化）

### 3.1 6 大品类
| 品类 | 示例 | 数量目标 | 解锁方式 |
|------|------|----------|----------|
| **像素萌宠** | 像素熊猫/柴犬/刺猬/小狐狸 | 12 只 | 专注台随机出现 |
| **公考名师头像** | 袁东老师/某笔老师/某图老师头像 | 8 位 | 答 10 道真题解锁 |
| **申论金句卡** | 习总书记讲话/政府工作报告金句 | 20 句 | 申论得分 80+ 解锁 |
| **面试题型徽章** | 人际关系/组织管理/应急应变... | 7 枚 | 完成对应题型首次答题 |
| **时政热点卡** | 江苏省情/最新政策解读 | 15 张 | 阅读当日要闻 3 天 |
| **装饰背景** | 像素风首页主题背景 | 6 套 | 收集齐 1 套萌宠解锁 |

### 3.2 稀有度映射
| 稀有度 | 比例 | 主要品类 | 视觉 |
|--------|------|----------|------|
| 普通 | 70% | 时政热点卡、装饰券 | 灰底白字 |
| 稀有 | 23% | 面试题型徽章、申论金句卡 | 蓝底 |
| 史诗 | 6% | 公考名师头像、装饰背景 | 紫底 |
| 传说 | 1% | 限定版像素萌宠（季节限定）| 金底闪光 |

### 3.3 「限定」的玩法
- **季节限定**：每年 1/4/7/10 月各推出 1 只限定萌宠（如 7 月「暑期学习小熊猫」）
- **活动限定**：江苏省考出公告前后 1 周，推「备考加油包」
- **周年限定**：小程序上线周年庆推 1 套专属

## 四、收藏图鉴（社交+成就）

### 4.1 入口与展示
- 「我的」→「补给站」按钮 → 进入主页面
- 主页面顶部：当前学习点数 + 收集进度 `24/68`
- 主页面下方：网格展示 6 个品类的图标（未收集的显示「?」灰色）

### 4.2 单品类详情
点击任一品类 → 进入该品类列表（参考截图2的「奖池」）：
- 横向滚动展示该品类所有补给品
- 已收集：彩色 + 收集日期
- 未收集：灰色 + 文字「??」

### 4.3 分享与炫耀
- 单个补给品可生成**像素风卡片**（小程序码 + 补给品图 + 收集日期）
- 卡片支持保存到相册 / 分享到微信好友
- 分享出去后**对方**可获得免费 1 抽（埋点：分享转化率）

### 4.4 图鉴成就
- **收集齐 1 套**（如全部 12 只萌宠）：解锁限定「成就徽章」+ 1000 学习点奖励
- **稀有度成就**：「集齐全部 8 位公考名师头像」+ 永久称号「公考百晓生」
- **进度可视化**：图鉴页面顶部一个「收集进度条」，按总收集数百分比

## 五、数据模型（Prisma Schema 草案）

```prisma
// 用户学习点余额（独立表，不绑定 User 主体表，方便扩展）
model UserPoints {
  userId      String   @id
  points      Int      @default(0)
  totalEarned Int      @default(0)   // 累计获得（用于成就）
  totalSpent  Int      @default(0)   // 累计消耗
  updatedAt   DateTime @updatedAt
  user        User     @relation(fields: [userId], references: [id])
}

// 补给品定义（系统侧，不变）
model SupplyItem {
  id          Int      @id @default(autoincrement())
  category    String   // pixelPet / teacherAvatar / shenlunQuote / examBadge / newsCard / theme
  rarity      String   // common / rare / epic / legendary
  name        String
  description String?
  imageUrl    String
  unlockHint  String?  // "完成 10 道真题"
  isLimited   Boolean  @default(false)
  validFrom   DateTime?
  validTo     DateTime?
  createdAt   DateTime @default(now())
}

// 用户收集记录
model UserCollection {
  id        Int      @id @default(autoincrement())
  userId    String
  itemId    Int
  source    String   // dailyFree / focus25 / answerQuestion / shareReward
  obtainedAt DateTime @default(now())
  isShared  Boolean  @default(false)
  user      User     @relation(fields: [userId], references: [id])
  item      SupplyItem @relation(fields: [itemId], references: [id])
  
  @@unique([userId, itemId])  // 同一补给品每人只收集 1 次
  @@index([userId, obtainedAt])
}

// 学习点流水（审计 + 反作弊）
model PointsLog {
  id        Int      @id @default(autoincrement())
  userId    String
  amount    Int      // 正为获得，负为消耗
  type      String   // answerComplete / focusComplete / drawSpend / drawReward / shareReward
  refId     String?  // 关联的答题/专注/抽奖 ID
  createdAt DateTime @default(now())
  
  @@index([userId, createdAt])
}
```

### 5.1 设计权衡
- **UserCollection 用 `@@unique([userId, itemId])`**：同补给品每人只能抽到 1 次（重复获得自动 +2 学习点）
- **重复抽到怎么办**：弹窗「已拥有！自动 +2 学习点」—— 避免浪费
- **PointsLog 流水表**：方便反作弊和客服查询

## 六、API 设计

| 路由 | 方法 | 用途 |
|------|------|------|
| `/api/supply/balance` | GET | 当前学习点余额 |
| `/api/supply/draw` | POST | 单次抽奖（参数：`{ source: 'free' \| 'paid' }`）|
| `/api/supply/collection` | GET | 我的图鉴（参数：`category?`）|
| `/api/supply/items/:id/share` | POST | 标记已分享（用于分享奖励）|
| `/api/supply/focus-reward` | POST | 专注完成后发放学习点（防作弊：需传实际专注时长）|

### 6.1 反作弊要点
- `focus-reward` 需校验：开始时间戳 + 当前时间戳 ≥ 设定时长，且中途未退出 5 分钟以上
- `draw` 限流：单用户最多 30 次/分钟（防脚本）
- 答题奖励：复用现有 `completeDailyTask` 触发点，避免重复发放

## 七、UI/UX 设计要点

### 7.1 视觉语言（与现有对齐）
- **背景**：`#f5f5f5`（与首页一致）
- **卡片**：白底 + `0 2px 12px rgba(0,0,0,0.04)` 阴影 + 16px 圆角
- **稀有度配色**：普通灰/稀有蓝 `#3b82f6`/史诗紫 `#9333ea`/传说金 `#f59e0b`
- **像素风**：所有补给品图保持 `image-rendering: pixelated`

### 7.2 关键页面
| 页面 | 路径 | 复杂度 |
|------|------|--------|
| 专注台 | `/subpkg-focus/pages/timer/index` | 中（番茄钟 + 倒计时）|

## 八、实施路径（分 4 阶段）

### Phase 1：最小可用版本（MVP，~1 周）
- 数据模型 + 后端 API（`/api/supply/balance` + `/api/supply/draw` + 4 个奖励发放点）
- 4 个品类共 20 个补给品（先不上名师头像，避免法律风险）
- 补给站抽奖页 + 简单动画
- 我的图鉴（仅展示）

**验收**：能答题获得学习点 → 抽到补给品 → 在图鉴看到

### Phase 2：专注台 + 分享（~1 周）
- 专注台番茄钟页面
- 分享卡片生成（canvas 绘制）
- 免费抽 + 分享奖励机制

**验收**：能专注 25 分钟 → 获得学习点 → 抽 → 分享到微信 → 对方获得免费抽

### Phase 3：完整品类 + 限定系统（~2 周）
- 公考名师头像品类（8 位，需用户确认法律风险）
- 申论金句卡 / 时政热点卡 / 面试题型徽章（动态内容）
- 季节限定系统
- 图鉴成就系统

**验收**：6 大品类完整上线，限定萌宠可获取

### Phase 4：埋点 + 调优（持续）
- 关键埋点：每日抽奖次数、品类收集率、分享转化率、停留时长
- A/B 测试不同概率下的留存变化
- 邀请码会员的学习点加成（如邀请用户 +20%/次）

## 九、风险与边界

### 9.1 风险点
| 风险 | 应对 |
|------|------|
| **未成年人沉迷抽奖** | 单日抽奖上限 50 次；未成年人模式禁用 |
| **氪金化倾向** | 学习点**只通过学习和分享获得**，不开放充值入口 |
| **抽到的补给品有版权风险** | 公考名师头像需**逐个授权**，先不上名人头像 |
| **学习点成为「第二货币」干扰额度制** | 学习点**只能换补给品**，不能换额度/会员 |
| **刷分**（脚本刷答题）| 复用现有反作弊 + 答题时长校验 |

### 9.2 明确不做的事
- ❌ **不做现金提现/充值入口**（避免被认定为赌博）
- ❌ **不做实时 PVP 答题对战**（与专注+收藏调性不符）
- ❌ **不做虚拟货币交易市场**（玩家不能交易补给品）
- ❌ **不做强制社交**（分享是奖励手段，不是门槛）

## 十、待用户确认的核心策略问题

以下 5 个问题直接影响数据模型和 API 设计，请先拍板再写代码：

1. **是否启用「公考名师头像」品类？**
   - 涉及肖像权风险。若启用需先做授权流程。

2. **学习点是否与现有额度打通？**
   - 方案 A：完全隔离（学习点只换补给品，额度仍是额度）—— **推荐**
   - 方案 B：学习点可兑换额度（1 学习点 = 0.01 额度，100 学习点换 1 次额度）

3. **免费抽政策？**
   - 方案 A：每日 1 次免费普通抽（推荐）—— 轻量但有惊喜
   - 方案 B：每周 3 次免费稀有抽 —— 更慷慨但成本高

4. **「分享得抽」的对方奖励？**
   - 方案 A：分享后**对方**获得免费 1 抽（裂变属性强）—— **推荐**
   - 方案 B：分享后**双方**各获得 50 学习点（更公平）

5. **优先做哪个 Phase？**
   - 方案 A：完整做完 Phase 1 再上线
   - 方案 B：Phase 1 + 专注台（Phase 1+2 合并）一起上
   - 方案 C：先做 1 周的 A/B 测试版本（只对 50% 用户开放）

## 十一、验收标准（Phase 1 完整版）

- [ ] 用户答题完成后 1 秒内看到「获得学习点 +1」提示

- [ ] 3 学习点可成功抽 1 次，动画流畅

- [ ] 反作弊：同一用户 1 分钟内最多 30 次抽奖请求
- [ ] 视觉与首页设计语言一致（`#f5f5f5` 背景 + 白卡 + 像素图标）
