# 小熊猫每日任务需求文档 v2

## 1. 需求背景

首页 Hero 卡片使用像素风小熊猫作为学习陪伴 mascot。为增强学习动力，将 mascot 与每日任务进度绑定。用户可自行设定每日目标数，熊猫图标按完成比例（0-33%/33-66%/66%+）在 3 张状态图之间切换，下方展示进度条与文案。

## 2. 素材清单

| 完成比例 | 文件名 | 含义 |
|----------|--------|------|
| 0% ~ 32% | `panda-reading-120.png` | 小熊猫阅读（初始/起步状态） |
| 33% ~ 65% | `panda-writing-120.png` | 小熊猫写试卷（进行中） |
| 66% ~ 100%+ | `panda-thumbsup-120.png` | 小熊猫点赞（接近完成/已完成） |

素材已处理：去除白底与右下角水印，输出 120/160/200 三个尺寸，统一存放到 `src/assets/images/`。

## 3. 任务规则

### 3.1 每日目标

- **默认值**：3 次。
- **可自定义**：用户可在首页实时修改目标数，范围 1–10 次。
- **练习类型**：面试答题、申论作答、套题训练均可计入。
- **每日重置**：以本地日期为准，跨天后自动重置完成次数为 0（目标数保留不变，不随日期重置）。

### 3.2 状态切换（按完成比例）

| 比例区间 | Mascot | 说明 |
|----------|--------|------|
| 0% ≤ progress < 33% | 阅读小熊猫 | 刚起步 |
| 33% ≤ progress < 66% | 写试卷小熊猫 | 认真练习中 |
| 66% ≤ progress ≤ 100% | 点赞小熊猫 | 接近完成 / 已完成 |

- **封顶**：当日完成次数最多累加到与目标数相等，超过不记。
- **重复提交**：同一次练习的重复提交不计入。
- **进度 = count / target × 100%**（取整）。

### 3.3 不同目标下的进度变化示例

| 目标 | 0 次 | 1 次 | 2 次 | 3 次 | 4 次 | 5 次 | 6 次 |
|------|------|------|------|------|------|------|------|
| **1** | 📖 0% | 👍 100% | ← | ← | ← | ← | ← |
| **2** | 📖 0% | ✍ 50% | 👍 100% | ← | ← | ← | ← |
| **3** | 📖 0% | ✍ 33% | 👍 67% | 👍 100% | ← | ← | ← |
| **4** | 📖 0% | 📖 25% | ✍ 50% | 👍 75% | 👍 100% | ← | ← |
| **5** | 📖 0% | 📖 20% | ✍ 40% | ✍ 60% | 👍 80% | 👍 100% | ← |
| **10** | 📖 0% | 📖 10% | 📖 20% | 📖 30% | ✍ 40% | ✍ 50% | ✍ 60% |

> 📖 = reading, ✍ = writing, 👍 = thumbsup  
> ← 表示超出目标，不再增加

## 4. 首页表现

### 4.1 Mascot 区域

- 根据完成比例自动切换对应小熊猫图片。
- 图片尺寸使用 100px（与当前首页一致）。
- 图片右侧保留「连续打卡 N 天」「今日练习 N」「今日目标 N%」等紧凑数据。

### 4.2 目标设置入口

- 在「今日目标」数字右侧放置一个小的编辑图标（⚙ 或 ✎），颜色 `#9ca3af`，字号 14px。
- 点击后弹出目标设置面板：
  - 半屏弹层（bottom sheet），背景白色，顶部圆角 20px。
  - 标题「设置今日目标」。
  - 预设按钮：1 次、3 次、5 次、10 次，以横向 flex 排列。
  - 或提供 + / - 步进器，范围 1–10。
  - 底部「确认」按钮保存并关闭。
  - 修改目标后，进度即刻重新计算并刷新界面。
- 目标设置仅在登录后可用，未登录时不展示编辑图标。

### 4.3 进度条

- 位于小熊猫图片正下方。
- 进度 = `已完成次数 / 目标次数 × 100%`。
- 高度 6px，蓝色渐变 `linear-gradient(90deg, #3b82f6, #60a5fa)`。
- 完成时进度为 100%，可配合「已完成」提示。

### 4.4 文案

| 条件 | 文案 |
|------|------|
| 未开始（count = 0） | 今日目标：完成 N 次练习 |
| 进行中（0 < count < target） | 还差 X 次完成今日目标 |
| 已完成（count ≥ target） | 今日目标已完成！ |

其中 `N` 为当前目标数，`X = target - count`。

## 5. 状态持久化

### 5.1 本地存储字段

使用 `Taro.setStorageSync` / `Taro.getStorageSync` 保存：

| Key | 类型 | 说明 |
|-----|------|------|
| `daily_task_date` | `string` | 最后记录日期 `YYYY-MM-DD` |
| `daily_task_count` | `number` | 当日已完成次数 0–target |
| `daily_task_target` | `number` | 每日目标数 1–10，默认 3 |

### 5.2 每日重置逻辑

首页加载时：

1. 读取 `daily_task_date`。
2. 若不等于今天：重置 `daily_task_count = 0`，更新 `daily_task_date` 为今天。`daily_task_target` 保持不变。
3. 读取 `daily_task_target`，若无则默认 3。

### 5.3 完成任务逻辑

调用 `completeDailyTask()` 时：

1. 获取当前日期，必要时重置。
2. 获取当前 target。
3. 若 count < target，则 count + 1。
4. 写回 storage。
5. 返回最新状态 `{ count, target, completed, progress, date }`。

### 5.4 修改目标逻辑

调用 `setDailyTaskTarget(n)` 时：

1. 更新 `daily_task_target = n`。
2. 若 count > n 则裁剪 count = n。
3. 写回 storage。
4. 返回最新状态。

## 6. 计数触发点

以下操作成功完成后调用 `completeDailyTask()`：

1. 面试真题语音/文字答题提交成功（`subpkg-zhenti` / `subpkg-practice` 相关页面）。
2. 申论真题作答提交成功（`subpkg-shenlun` 相关页面）。
3. 套题训练完成（`subpkg-practice` 套题生成/训练页面）。

**注意**：需要在提交成功的回调中调用，避免提交失败也计入次数。

## 7. 边界情况

- **未登录用户**：仍使用本地 storage 记录进度，但目标设置入口不显示，使用默认目标 3。
- **跨天使用**：首页加载时自动重置完成次数，但目标数不重置。
- **超过目标**：count 保持 target，不再增加，mascot 保持点赞状态。
- **修改目标为更小值**：若已完成数大于新目标，自动裁剪至新目标。
- **日期获取失败**：以 0 次 / 默认目标 3 处理。
- **目标 = 1**：0 次 → 阅读，1 次 → 立即点赞（无写作阶段），符合用户预期。

## 8. 技术实现

### 8.1 修改文件

| 文件 | 改动 |
|------|------|
| `src/utils/dailyTask.ts` | 新增 `setDailyTaskTarget(n)`；`getDailyTask()` 返回新增 `target` 字段；`completeDailyTask()` 改用 target 判断上限；进度按 target 计算 |
| `src/pages/index/index.tsx` | 新增目标设置面板（弹层 + 预设按钮）；在「今日目标」旁边显示编辑图标；mascot 改为按比例切换 |
| `src/pages/index/index.scss` | 新增目标设置面板样式（弹层、预设按钮、步进器） |

### 8.2 不改动部分

- 4 个练习完成页面的 `completeDailyTask()` 调用保持不变（API 签名兼容）。
- 3 张小熊猫素材不变。
- 首页 Hero 卡片整体布局不变。

## 10. 验收标准

- [ ] 首页默认目标为 3 次，初始显示阅读 panda + 0% 进度。
- [ ] 点击「今日目标」旁的编辑图标可打开目标设置面板。
- [ ] 设置面板可选择 1/3/5/10 次预设或手工输入 1-10。
- [ ] 修改目标后进度和 panda 即刻更新。
- [ ] 完成比例 0-32% → 阅读 panda；33-65% → 写作 panda；66-100% → 点赞 panda。
- [ ] 目标 = 3 时：0 次→阅读，1 次→写作（33%），2 次→点赞（66%）。
- [ ] 跨天后完成次数重置，目标数不变。
- [ ] 超过目标后 count 不再增加，mascot 保持点赞。
- [ ] 4 个练习完成点调用 `completeDailyTask()` 正常触发。
- [ ] `npm run build:weapp` 编译通过。
