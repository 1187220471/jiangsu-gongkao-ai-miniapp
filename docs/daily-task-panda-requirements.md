# 小熊猫每日任务需求文档

## 1. 需求背景

首页 Hero 卡片使用像素风小熊猫作为学习陪伴 mascot。为增强学习动力，将 mascot 与每日任务进度绑定：用户每天完成 3 次练习后，mascot 会切换为庆祝状态，并在 mascot 下方展示任务进度条。

## 2. 素材清单

| 完成次数 | 文件名 | 含义 |
|----------|--------|------|
| 0 / 3 | `panda-reading-120.png` | 小熊猫阅读（初始状态） |
| 1-2 / 3 | `panda-writing-120.png` | 小熊猫写试卷（进行中） |
| 3 / 3 | `panda-thumbsup-120.png` | 小熊猫点赞（完成状态） |

素材已处理：去除白底与右下角水印，输出 120/160/200 三个尺寸，统一存放到 `src/assets/images/`。

## 3. 任务规则

- **每日目标**：3 次有效练习。
- **练习类型**：面试答题、申论作答、套题训练均可计入。
- **每日重置**：以本地日期为准，跨天后自动重置为 0/3。
- **状态切换**：
  - 0 次完成：显示阅读小熊猫。
  - 完成 1 次：切换为写试卷小熊猫。
  - 完成 3 次：切换为点赞小熊猫。
- **封顶**：当日完成次数最多记 3 次，超过不再累加，mascot 保持点赞状态。
- **重复提交**：同一次练习的重复提交不计入。

## 4. 首页表现

### 4.1 Mascot 区域

- 根据当前完成次数显示对应小熊猫图片。
- 图片尺寸使用 120px（与当前首页一致）。
- 图片右侧保留「连续打卡 N 天」「今日练习 N」「今日目标 N%」等紧凑数据。

### 4.2 进度条

- 位于小熊猫图片正下方。
- 进度 = `已完成次数 / 3 * 100%`。
- 样式与现有进度条一致（高度 6px，蓝色渐变）。
- 完成时进度为 100%。

### 4.3 文案

| 完成次数 | 主文案 | 进度文案 |
|----------|--------|----------|
| 0 / 3 | 今日目标：完成 3 次练习 | 0% |
| 1 / 3 | 今日目标：完成 3 次练习 | 33% |
| 2 / 3 | 还差 1 次完成今日目标 | 67% |
| 3 / 3 | 今日目标已完成！ | 100% |

## 5. 状态持久化

### 5.1 本地存储字段

使用 `Taro.setStorageSync` / `Taro.getStorageSync` 保存：

- `daily_task_date`：最后记录日期，格式 `YYYY-MM-DD`。
- `daily_task_count`：当日已完成次数，整数 0-3。

### 5.2 每日重置逻辑

首页加载时读取 `daily_task_date`：

- 若等于今天：使用 `daily_task_count`。
- 若不等于今天：重置 `daily_task_count = 0`，并更新 `daily_task_date` 为今天。

### 5.3 完成任务逻辑

调用 `completeDailyTask()` 时：

1. 获取当前日期。
2. 若存储日期不是今天，重置 count = 0 并更新日期。
3. 若 count < 3，则 count + 1。
4. 写回 storage。
5. 返回最新状态 `{ count, total, completed, date }`。

## 6. 计数触发点

以下操作成功完成后调用 `completeDailyTask()`：

1. 面试真题语音/文字答题提交成功（`subpkg-zhenti` / `subpkg-practice` 相关页面）。
2. 申论真题作答提交成功（`subpkg-shenlun` 相关页面）。
3. 套题训练完成（`subpkg-practice` 套题生成/训练页面）。

**注意**：需要在提交成功的回调中调用，避免提交失败也计入次数。

## 7. 边界情况

- **未登录用户**：仍使用本地 storage 记录进度，登录后进度保留（V2 可考虑与后端同步）。
- **跨天使用**：首页加载时自动重置。
- **超过 3 次**：count 保持 3，不再增加。
- **日期获取失败**：以 0 次处理，不影响首页展示。

## 8. 技术实现

### 8.1 新增文件

- `src/utils/dailyTask.ts`：每日任务状态管理（读取、重置、完成）。
- `docs/daily-task-panda-requirements.md`：本文档。

### 8.2 修改文件

- `src/pages/index/index.tsx`：
  - 引入 3 张 panda 图片；
  - 使用 `getDailyTask()` 获取状态；
  - 根据 count 选择 mascot；
  - 更新进度条和文案。
- `src/pages/index/index.scss`：进度条与 mascot 下方间距保持 6-8px。
- 各练习完成页面：在提交成功后调用 `completeDailyTask()`。

## 9. 验收标准

- [ ] 首页初始显示阅读小熊猫 + 0% 进度。
- [ ] 完成 1 次练习后显示写试卷小熊猫 + 33% 进度。
- [ ] 完成 2 次练习后显示写试卷小熊猫 + 67% 进度。
- [ ] 完成 3 次练习后显示点赞小熊猫 + 100% 进度。
- [ ] 完成 3 次后继续练习，count 保持 3，mascot 保持点赞。
- [ ] 跨天后首页自动重置为阅读小熊猫 + 0%。
- [ ] `npm run build:weapp` 编译通过。
